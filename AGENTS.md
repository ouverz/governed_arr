# Repository Instructions

## Project Context

When starting with no project context, or when the available context is
insufficient to answer or act confidently, first read these files in order:

1. `README.md`
2. `docs/project_walkthrough.md`

Use them to understand the project's purpose, architecture, implementation
sequence, important decisions, limitations, and expected results before
continuing.

Read additional source files only as needed for the specific task. Verify claims
against the implementation rather than relying only on the overview documents.

## Critical Review Before Important Responses

Before providing any important response, critically review the proposed answer
or completed work.

Check that:

- factual claims are supported by the repository or clearly labeled as
  assumptions;
- the response answers the user's actual request;
- important risks, limitations, inconsistencies, and trade-offs are surfaced;
- code changes follow existing project patterns and do not introduce unrelated
  changes;
- tests or verification performed are reported accurately;
- tests or verification not performed are stated explicitly;
- recommendations are technically defensible and not based on reflexive
  agreement with the user's premise; and
- the final response is clear, concise, and does not overstate confidence.

Correct issues found during this review before giving the response. For
substantial work, inspect the final diff or relevant output as part of the
review.
